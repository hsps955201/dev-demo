# Guidelines for Git Commit Messages and PRs

To maintain clarity in our commit history, it is advisable to use a consistent format for commit messages. A well-structured commit message typically includes:

1. **Ticket Number**: Please reference the related issue.
2. **Verb**: It is recommended to start with an imperative verb.
3. **Description**: Kindly provide a brief statement of what was accomplished.

### Example Messages

- `#123: fix: Update README.md formatting`
